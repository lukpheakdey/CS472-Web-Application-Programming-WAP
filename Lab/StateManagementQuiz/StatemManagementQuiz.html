State Management Quiz

Question 1: Can post requests be bookmarked? What are the problems?

NO, Post request cannot be bookmarked because the data sent to the server with POST is stored in the request body of the HTTP request.

Question 2: What is the purpose of request dispatching?

The purpose of request dispatching is to internally pass the request processing to another local resource or to tell client browser to issue another HTTP request to a specific URL.

Question 3: What is the difference between redirect and request dispatch?

Forward:
  • Pass the request to another resource on the server. Sometimes referred as "server side redirect".
  • The request and response object passed to the destination servlet.
  • Browser is completely unaware of servlet forward, and hence the the URL will remain unchanged.
Redirect:
  • Servlet sends HTTP code 3xx to client along with the redirect URL (usually 302 temporary redirect).
  • Client then sends a new request to the URL.
  • extra round trip.
  • only HTTP message sent, the request and response object cannot be sent.

Question 4: What is an attribute ?

An attribute is an object that is used to share information in a web app. Attribute allows servlet to share information among themselves.
An attribute is an object bound into one of three servlet API objects:
  • HTTPServletRequest.
  • HTTPSession.
  • ServletContext.

Question 5: What is the difference between attributes and parameters?

  • Parameter is a name/value pair sent from client to the server - typically, from a HTML form. Parameters can only store String values.
  • An attribute is a server-local storage mechanism. Attributes have String name, but store Object values. Note that attributes are specific to Java, while parameter are platform-independent.

Question 6: What are dangers of using attributes?

Shared by every servlet and every request in the application.

Question 7: What does it mean to say that http is stateless? Give an example of a stateful protocol.

HTTP is a stateless protocol, which means that after a web server responds to a request, there is no "memory" of the request or who sent it.
A protocal which requires keeping of the internal state on the server is known as a stateful protocol, example: TCP connection-oriented session.

Question 8: Give 5 different methods for maintaining state information (count each attribute scope as one method).

  • Request scope: destroyed when servlet finishes processing request.
  • Session scope: destroyed when user closes browser.
  • Application scope: destroyed when container stopped.
  • Cookies saved on browser: temporary (deleted when the browser closes), permanent.
  • Hidden field on a form.

Question 9: How long does a session last?

There are 3 ways to remove sessions:
  • session timeout in the DD.
  • using method: session.setMaxInactiveInterval(time)
  • using method: session.invalidate()

Question 10: What is a cookie, and how long does a cookie last?

Cookie is small data which is stored on a client side. It is designed to hold a modest amount of data specific to a particular client and website, and can be accessed either by the web server or the client computer.
There are two types of cookie:
  • temporary cookie
  • browser removes when it closes
  • this is default
  • Permanent cookie: a cookie that has a max age set.

Question 11: What is the purpose of URL rewriting?

URL rewriting is used to provide shorter and more relevant-looking links to web pages.
